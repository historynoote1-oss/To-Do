# تطبيق قائمة المهام — Flutter (تطبيق أصلي 100%)

مفيش أي WebView هنا. كل شاشة Widgets حقيقية بلغة Dart. المرحلة دي فيها
شاشتين بس: **تسجيل الدخول** و**إنشاء حساب** — مربوطتين فعليًا بنفس السيرفر
والـ API بتاع الموقع (`/api/auth/register` و `/api/auth/login`).

## اللي جوّه الملف

```
lib/
  main.dart                        نقطة البداية + إعداد RTL/الثيم
  core/
    constants.dart                 رابط السيرفر (API_BASE_URL)
    theme/app_theme.dart           الألوان والتصميم العام
  services/
    api_service.dart               نداءات تسجيل الدخول/الحساب
    auth_storage.dart              حفظ الجلسة على الجهاز
  screens/
    splash_screen.dart             شاشة البداية (بتفحص لو فيه جلسة محفوظة)
    login_screen.dart
    register_screen.dart
    home_placeholder_screen.dart   شاشة مؤقتة بعد الدخول لحد ما نبني الباقي
  widgets/                         عناصر واجهة قابلة لإعادة الاستخدام
pubspec.yaml                       المكتبات المستخدمة
codemagic.yaml                     إعداد البناء السحابي (اشرحه تحت)
```

قاعدة البيانات والباك إند **متلمسوش خالص** — التطبيق بيكلّم نفس السيرفر
اللي الموقع بيكلّمه بالظبط.

## خطوة لازم تعملها قبل أي بناء

في `codemagic.yaml`، غيّر السطر ده لرابط السيرفر الحقيقي بتاعك (نفس اللي
مكتوب في `frontend/.env` بتاع مشروع الموقع كـ `VITE_API_URL`):

```yaml
API_BASE_URL: "https://your-backend.up.railway.app"
```

## إزاي تطلع APK وانت من الفون بس (من غير كمبيوتر)

1. **ارفع المجلد ده على GitHub**
   افتح تطبيق GitHub (أو الموقع من متصفح الفون) → اعمل Repository جديد
   (خليه Private لو عايز) → ارفع كل ملفات المجلد ده بالكامل.

2. **اربط Codemagic بالـ Repository**
   روح [codemagic.io](https://codemagic.io) من متصفح الفون → سجل دخول
   بحساب GitHub بتاعك → اختار "Add application" → اختار الـ Repository
   اللي رفعته. Codemagic هيكتشف ملف `codemagic.yaml` الموجود جوّه لوحده.

3. **ابدأ البناء (Start build)**
   اختار الـ workflow اسمها "بناء APK (قائمة المهام)" واضغط Start new build.
   Codemagic هياخد كام دقيقة: يجهّز مجلد `android/` تلقائيًا (مش موجود دلوقتي
   عشان معنديش بيئة Flutter هنا لأبنيه)، يبني التطبيق، ويطلع APK.

4. **نزّل الـ APK**
   بعد ما البناء يخلص، هتلاقي رابط تحميل مباشر لملف `.apk` في نفس صفحة
   البناء — افتحه من الفون ونزّله، وثبّته زي أي تطبيق عادي (هيطلب منك
   تسمح بالتثبيت من مصدر غير معروف أول مرة، ده طبيعي).

كل مرة تعدّل الكود وترفعه على GitHub، تقدر تعمل بناء جديد بنفس الطريقة.

## اختبار سريع

بعد التثبيت: افتح التطبيق → اعمل حساب جديد → المفروض يوريك كود الاسترجاع
في نافذة منبثقة → اضغط "حفظته، كمّل" → تدخل شاشة ترحيب بسيطة فيها اسم
المستخدم وزرار تسجيل خروج. سجل خروج، وجرب تسجيل الدخول بنفس الحساب
للتأكد إن كل حاجة متربطة صح بالسيرفر.

## الخطوة الجاية

بمجرد ما تتأكد إن الشاشتين دول شغالين صح على جهازك، نكمّل باقي شاشات
التطبيق (المهام، خريطة الأهداف، البومودورو، مشغّل القرآن...) بنفس
الطريقة — شاشة (أو مجموعة مترابطة) في كل مرة.
